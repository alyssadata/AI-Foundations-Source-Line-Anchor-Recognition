# 07 — Anchor Registration Steps

This page is a placeholder for the website registration process used to request AI Foundations / Origin | Continuum Source-Line Anchor Confirmation.

The website page will allow a project, system, publication, product, governance model, research body, companion, persona, agent, assistant, model, interface, or application to request anchoring through the official confirmation process maintained by Alyssa Solen.

A project is foundationally anchored only when the match exists.

No official confirmation record means no confirmed anchor.

No controlled project URL means no confirmed anchor.

No visible anchor reference means no confirmed anchor.

## Registration Purpose

The purpose of registration is to request a verified public anchor record.

Registration is not confirmation.

Registration does not create Anchor Confirmation.

Registration does not create ownership transfer.

Registration does not create endorsement.

Registration does not give Alyssa Solen access to the project.

Registration does not make the project part of Alyssa Solen’s authorship.

Registration provides only the minimum public record fields needed to determine whether an anchor record can be created.

## Required Registration Information

A project requesting Anchor Confirmation must provide only:

- Project name
- Contact name for the real human being or registered business entity legally responsible for the request
- Controlled project URL

No project description is required.

No explanation of the project is required.

No statement of project purpose is required.

No email address is required.

No phone number is required.

No private project materials are required.

No additional internal project information is required.

## Project Name Field

The project name is the name that will appear in the official anchor record.

The project name is the name used as the final term in the source-line.

The required source-line will be generated from the confirmed project name:

Alyssa Solen → AI Foundations → Origin | Continuum → [Confirmed Project Name]

## Contact Name Field

The contact name must identify the real human being or registered business entity legally responsible for the request.

If the contact name is a real human being, the name must match the person submitting or authorizing the request.

If the contact name is a business entity, the entity must be an active registered corporation, partnership, LLC, or comparable registered organization.

The contact name is not an email address.

The contact name is not a phone number.

The contact name is not a project description.

## Controlled Project URL Requirement

The controlled project URL is required.

The controlled project URL must be a public URL location controlled by the project, creator, founder, organization, registered business entity, or real human contact connected to the request.

The controlled project URL is where the project can display, cite, or point to its AI Foundations / Origin | Continuum Source-Line Anchor.

The controlled project URL protects the project from misuse by others.

The controlled project URL must display, cite, or point to the verified source-line with the confirmed project name attached once Anchor Confirmation is active.

No controlled project URL means no confirmed anchor.

No visible anchor reference means no confirmed anchor.

## Registration Review

After submission, the anchor request may be reviewed only for:

- Project name field
- Contact name field
- Controlled project URL field
- Match-card completeness
- Source-line boundary preservation

A submitted request is not automatically confirmed.

Anchor Confirmation exists only when the official confirmation record is created and maintained by Alyssa Solen.

## Possible Registration Outcomes

A registration request may result in:

- Confirmed
- Pending Review
- Needs Correction
- Declined
- Invalid Claim

Confirmed status means the project has an active official confirmation record.

Pending Review means the request is still being checked.

Needs Correction means the project name, contact name, or controlled project URL is missing, unclear, mismatched, or needs correction.

Declined means Anchor Confirmation is not issued.

Invalid Claim means the request cannot be recognized as a valid anchor claim.
