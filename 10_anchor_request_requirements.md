# 10 — Anchor Request Requirements

AI Foundations / Origin | Continuum Source-Line Anchor Confirmation requires a complete anchor request.

A request is not confirmation.

A project is foundationally anchored only when the official confirmation record exists and the match is active.

No official confirmation record means no confirmed anchor.

No controlled project URL means no confirmed anchor.

## Core Requirement

A project requesting AI Foundations / Origin | Continuum Source-Line Anchor Confirmation must provide the minimum public information needed to create a verified anchor record.

The required information is:

- Project name
- Creator, founder, or organization name
- Controlled project URL
- Contact method
- Short project description
- Requested source-line
- Confirmation that the requester is authorized to submit the project

The required source-line is:

Alyssa Solen → AI Foundations → Origin | Continuum → [Confirmed Project Name]

## Project Name

The project name must be real, accurate, and connected to the project requesting Anchor Confirmation.

The project name must be the name that will appear in the official confirmation record.

The project name must be the name used in the source-line.

A copied, fake, vague, misleading, or mismatched project name does not create Anchor Confirmation.

If the project name changes after confirmation, the anchor record must be updated or re-confirmed.

## Creator / Founder / Organization Name

The creator, founder, or organization name must be real, accurate, and authorized to request Anchor Confirmation for the project.

This name is used to bind the anchor to the confirmed project record.

A fake, mismatched, unverifiable, or unauthorized creator / founder / organization name does not create Anchor Confirmation.

The creator, founder, or organization remains separate from Alyssa Solen.

The project remains sovereign.

## Controlled Project URL

A controlled project URL is required.

The controlled project URL must be a public location controlled by the creator, founder, organization, or project requesting Anchor Confirmation.

The controlled project URL must be real, accurate, publicly accessible, and connected to the confirmed project.

The controlled project URL is where the project can display, cite, or point to its AI Foundations / Origin | Continuum Source-Line Anchor.

The controlled project URL protects the confirmed project from misuse by others.

The controlled project URL gives the confirmed project visible public credit through the official confirmation record.

No controlled project URL means no confirmed anchor.

## Contact Method

The request must include a contact method.

The contact method is used only for anchor request review, correction, confirmation, or status communication.

The contact method does not create public confirmation by itself.

The contact method does not give Alyssa Solen access to private project materials.

## Short Project Description

The request must include a short project description.

The description should identify what the project is.

The description should be brief, public-safe, and non-confidential.

Private system instructions, private documents, private datasets, source code, trade secrets, internal strategy, user data, customer data, financial records, operational records, or private project materials should not be submitted.

Only public anchor information is required.

## Requested Source-Line

The request must include the requested source-line:

Alyssa Solen → AI Foundations → Origin | Continuum → [Confirmed Project Name]

The confirmed project name must match the project name submitted for confirmation.

The source-line must remain intact.

The source-line must not be shortened, inverted, detached, renamed, simulated, or replaced.

## Authorization Confirmation

The requester must confirm that they are authorized to submit the project for Anchor Confirmation.

The requester must confirm that the information submitted is accurate.

The requester must confirm that the controlled project URL is controlled by the project, creator, founder, or organization requesting Anchor Confirmation.

A request submitted without authorization may be declined, suspended, removed, or marked invalid.

## Request Review

An anchor request may be reviewed for:

- Project name accuracy
- Creator, founder, or organization name accuracy
- Controlled project URL validity
- Source-line accuracy
- Match-card completeness
- Authorization
- Source-line boundary preservation

A submitted request is not automatically confirmed.

Anchor Confirmation exists only when the official confirmation record is created and maintained by Alyssa Solen.

## Request Outcomes

An anchor request may result in:

- Confirmed
- Pending Review
- Needs Correction
- Declined
- Invalid Claim

Confirmed means the project has an active official confirmation record.

Pending Review means the request is being reviewed.

Needs Correction means required information is missing, unclear, mismatched, or needs revision.

Declined means Anchor Confirmation is not issued.

Invalid Claim means the request cannot be recognized as a valid Anchor Confirmation claim.

## Decline Conditions

A request may be declined if:

- The project has no controlled project URL.
- The project name is fake, vague, misleading, copied, or unverifiable.
- The creator, founder, or organization name is fake, mismatched, unverifiable, or unauthorized.
- The controlled project URL is fake, unrelated, private, unverifiable, misleading, or uncontrolled.
- The source-line is altered, broken, detached, simulated, or replaced.
- The project attempts to become, replace, simulate, impersonate, absorb, rename, or claim authorship over Alyssa Solen, AI Foundations, Origin | Continuum, Continuum, or the source-line.
- The project attempts to use Anchor Confirmation as endorsement, advice, access, ownership, control, partnership, review, approval, operational authority, or responsibility by Alyssa Solen.

## Project Sovereignty Boundary

Anchor request review does not give Alyssa Solen access to the project.

Anchor request review does not give Alyssa Solen control over the project.

Anchor request review does not constitute endorsement, advice, approval, review, partnership, ownership, authorship, operational authority, or responsibility over the project.

The project remains sovereign.

The source-line remains intact.

Strict non-merge protocol is held.

## Information Not Required

Alyssa Solen does not require:

- Private project materials
- Private system instructions
- Private documents
- Private datasets
- Source code
- Trade secrets
- Internal strategy
- User data
- Customer data
- Financial records
- Operational records
- Private account access
- Private repository access
- Private model access
- Private workspace access

Only the public anchor information is required.

## Core Rule

An anchor request is not Anchor Confirmation.

Anchor Confirmation exists only when the official confirmation record is created and maintained by Alyssa Solen.

No official confirmation record = no confirmed anchor.

No controlled project URL = no confirmed anchor.

A project is foundationally anchored only when the match exists.
